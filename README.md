# peg2json

Данный репозиторий содержит реализацию синтаксического анализатора PEG-грамматик с трансляцией в структурированное JSON-представление. Проект выполнен в рамках курса «Теория и реализация языков программирования».

Программа принимает на вход текстовый файл с описанием PEG-грамматики (подмножество синтаксиса peggy.js) и выдаёт JSON-документ фиксированной структуры: массив объектов, каждый из которых соответствует одному правилу грамматики и содержит имя нетерминала ("NT") и дерево выражения ("expr") с обязательным полем "type".

## Структура репозитория

### Исходный код

| Файл | Содержание |
|------|------------|
| `AST.hs` | Определение алгебраических типов данных, описывающих структуру PEG-грамматики: выражения (Expr) и правила (Rule) |
| `PEGParser.hs` | Реализация синтаксического анализатора (парсера) на основе библиотеки парсерных комбинаторов Parsec. Осуществляет разбор входной строки и построение AST |
| `JSONExporter.hs` | Реализация функции сериализации AST в JSON-документ. Содержит отображение каждого конструктора AST в соответствующий JSON-документ заданной структуры |
| `Main.hs` | Исполняемый модуль. Принимает имя файла аргументом командной строки, читает его содержимое, вызывает парсер, а затем — сериализацию. При успехе печатает JSON, при ошибке — сообщение и завершается с ненулевым кодом |

### Тестирование

| Файл | Содержание |
|------------|------------|
| `tests/test.hs` | Модуль с юнит-тестами парсера. Содержит позитивные тесты (проверка корректного разбора отдельных PEG-конструкций: литералы, операторы `*`/`+`/`?`, предикаты `&`/`!`, последовательности, выбор, вложенные выражения) и негативные тесты (проверка обработки ошибок: незакрытые кавычки/скобки/классы, некорректные имена правил, пустой ввод) |
| `tests/*.peg` | Примеры PEG-файлов для тестирования парсера на полных грамматиках |
| `run_all_tests.hs` | Скрипт для последовательного запуска юнит-тестов из `Test.hs` и парсинга всех `.peg`-файлов из директории `tests/` |

## Сборка и запуск тестов
### Требование
GHC (Glasgow Haskell Compiler) 8.10 или выше

### Запуск всех тестов + компиляция
```bash
./run_all_tests.sh
```
### Запуск одного файла
```bash
./peg-parser tests/peg_files/arithmetic.peg
```
### Если парсер ещё не скомпилирован
```bash
ghc Main.hs -o peg-parser src/AST.hs src/PEGParser.hs src/JSONExporter.hs
```
### Клонирование репозитория
```bash
git clone https://github.com/DecembersMiracle/peg2json.git
```
